# 轻量桌面翻译器项目结构约定
## 1. 目标
在真正开始写代码前，先把目录、依赖方向和命名方式固定下来。

## 2. 推荐目录
```text
.
├── docs/
├── frontend/
│   ├── src/
│   │   ├── app/
│   │   ├── components/
│   │   ├── features/
│   │   ├── styles/
│   │   └── tests/
├── src-tauri/
│   ├── src/
│   │   ├── app/
│   │   ├── orchestrator/
│   │   ├── providers/
│   │   ├── http_api/
│   │   ├── storage/
│   │   ├── platform/
│   │   └── errors/
├── platform/
│   └── macos/
│       └── helper/
└── scripts/
```

## 3. 依赖方向
- `frontend` 可以调用 Tauri 暴露的 command，但不直接碰平台原生能力。
- `src-tauri/orchestrator` 可以依赖 `providers`、`storage`、`platform`。
- `providers` 不依赖 `frontend`。
- `platform` 不依赖具体 Provider。
- `http_api` 复用 `orchestrator`，不复制业务逻辑。

## 4. 命名约定
- React 组件：`PascalCase.tsx`
- React hooks：`useXxx.ts`
- Rust 文件：`snake_case.rs`
- 文档文件：中文专题名
- Provider ID：`snake_case`

## 5. 测试位置
- 前端测试放 `frontend/src/tests/`
- Rust 单元测试跟随模块
- Rust 集成测试放 `src-tauri/tests/`
- 平台 Helper 的验证脚本放 `platform/macos/helper/tests/`

## 6. 边界约定
- 不在 `frontend` 内写平台特判。
- 不在 `providers` 中读取 UI 配置。
- 不在 `platform` 中拼接第三方 HTTP 请求。
- 不把错误码散落到多个目录重复定义。

## 7. 代码风格约定
- 业务逻辑优先写在 Rust。
- React 层以展示和交互为主。
- 平台特有逻辑集中在 `platform/macos`。
- 共享模型优先单点定义，再由前后端映射。

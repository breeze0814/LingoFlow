# 轻量桌面翻译器配置模型
## 1. 目标
统一定义哪些配置进 Store，哪些密钥进 Keychain，以及配置版本如何迁移。

## 2. 配置分层
- `Store`：普通配置、UI 偏好、窗口状态、端口等非敏感信息。
- `Keychain`：API Key、Secret、Token。

## 3. Store 顶层结构
```json
{
  "version": 1,
  "app": {},
  "shortcuts": {},
  "providers": {},
  "http_api": {},
  "ui": {},
  "window": {}
}
```

## 4. 字段定义
### 4.1 app
```json
{
  "source_lang": "auto",
  "target_lang": "zh-CN"
}
```

### 4.2 shortcuts
```json
{
  "selection_translate": "Cmd+Shift+1",
  "input_translate": "Cmd+Shift+2",
  "ocr_recognize": "Cmd+Shift+3",
  "ocr_translate": "Cmd+Shift+4"
}
```

### 4.3 providers
```json
{
  "default_translate_provider": "openai_compatible",
  "enabled_translate_providers": ["openai_compatible"],
  "default_ocr_provider": "apple_vision",
  "enabled_ocr_providers": ["apple_vision"]
}
```

### 4.4 http_api
```json
{
  "enabled": true,
  "host": "127.0.0.1",
  "port": 61928
}
```

### 4.5 ui
```json
{
  "close_on_blur": true,
  "remember_window_bounds": true,
  "auto_copy_result": false
}
```

### 4.6 window
```json
{
  "main_panel": {
    "width": 520,
    "height": 420,
    "x": null,
    "y": null
  }
}
```

## 5. Keychain 命名约定
- `lingoflow.provider.openai_compatible.api_key`
- `lingoflow.provider.secondary_online_provider.api_key`

命名原则：
- 统一前缀
- 按 Provider 分隔
- 不把明文密钥写进 Store

## 6. 默认值原则
- 默认值必须可直接启动应用。
- 默认值必须保证至少一个翻译 Provider 和一个 OCR Provider 可选。
- 缺少云服务密钥时，应用可以启动，但相关 Provider 标记为未就绪。

## 7. 迁移规则
- `version` 为配置结构版本号。
- 任何破坏性修改都必须升级 `version`。
- 迁移逻辑只允许前进，不允许静默回退。
- 无法迁移时必须显式报错并保留原文件。

## 8. 不允许的实现
- 把 API Key 存进普通 JSON。
- 在多个地方维护不同的默认值。
- 让 Provider 自己读取配置文件。
- 用空字符串代表“有效配置”。

# 轻量桌面翻译器需求边界
## 1. 目标
这份文档只回答一个问题：
`V1 每个功能做到什么程度算完成。`

## 2. 全局边界
- `macOS` 为正式支持平台，`Windows` 为实验支持平台。
- V1 保留设置主窗口，并使用独立结果工作窗口承载输入、OCR 与翻译结果。
- V1 任意时刻只允许一个活跃任务。
- 新任务启动时，如果旧任务仍在 `pending`，旧任务必须先进入 `cancelled`。
- V1 默认突出展示一个首选结果，并保留 Provider 结果栈。
- V1 不做静默 fallback，不自动切到其他 Provider。

## 3. 划词翻译
### 3.1 输入
- 用户在任意前台应用中选中文本。
- 用户触发全局快捷键。

### 3.2 输出
- 主结果面板展示原文、译文、源语言、目标语言、当前 Provider。
- 用户可复制译文。

### 3.3 成功条件
- 成功读取到选中文本。
- 成功完成翻译请求。
- 面板展示最终结果。

### 3.4 失败条件
- 没有选中文本。
- 无法读取选中文本。
- 缺少辅助功能权限。
- 翻译 Provider 配置错误或请求失败。

### 3.5 非目标
- 不支持自动从剪贴板兜底。
- 不支持多段文本批量翻译。
- 不支持自动追加到历史长文本模式。

## 4. 输入翻译
### 4.1 输入
- 用户触发输入翻译快捷键。
- 用户在输入框内手动输入文本并提交。

### 4.2 输出
- 在结果工作窗口展示原文、译文和 Provider 结果栈。

### 4.3 成功条件
- 输入框可正常呼出。
- 用户提交后进入翻译流程。
- 成功返回译文并展示。

### 4.4 失败条件
- 输入内容为空。
- 当前默认翻译 Provider 不可用。
- 网络或配置错误导致翻译失败。

### 4.5 非目标
- V1 不做动态翻译。
- V1 不做草稿自动保存。
- V1 不做多输入会话并存。

## 5. 截图 OCR
### 5.1 输入
- 用户触发截图 OCR 快捷键。
- 用户框选屏幕区域。

### 5.2 输出
- 面板展示识别出的原文。
- 用户可复制识别结果。

### 5.3 成功条件
- 截图流程可正常启动。
- 成功拿到截图图像。
- OCR 返回可解析文本。

### 5.4 失败条件
- 用户取消截图。
- 缺少截图权限。
- OCR 返回空内容。
- OCR Provider 执行失败。

### 5.5 非目标
- V1 不做版面还原。
- V1 不做图片区域编辑。
- V1 不做公式、表格、手写专项识别。

## 6. 截图翻译
### 6.1 输入
- 用户触发截图翻译快捷键。
- 用户框选屏幕区域。

### 6.2 输出
- 面板展示 OCR 原文和翻译结果。

### 6.3 成功条件
- 截图成功。
- OCR 成功。
- 翻译成功。

### 6.4 失败条件
- 截图取消。
- OCR 失败。
- 翻译失败。
- 任一前置步骤失败导致链路中断。

### 6.5 非目标
- V1 不做图片内原位替换翻译。
- V1 不做图片排版重绘。
- V1 不做多区域连续截图翻译。

## 7. 本地 HTTP 调用
### 7.1 输入
- 其他本地程序通过 `127.0.0.1` 发起请求。

### 7.2 输出
- 对直接任务返回结构化 JSON。
- 对仅触发 UI 的命令返回 `accepted`，并实际打开对应工作区。

### 7.3 成功条件
- 本地服务成功启动。
- 路由能够命中。
- 请求得到明确响应结果。

### 7.4 失败条件
- 端口被占用。
- 参数不合法。
- 对应任务被用户取消。
- 调用链路中的 Provider 或权限失败。

### 7.5 非目标
- V1 不支持远程网络访问。
- V1 不支持 WebSocket 推送。
- V1 不支持插件回调。

## 8. 设置页
### 8.1 V1 必须具备
- 快捷键设置
- 默认目标语言设置
- 翻译 Provider 启用与默认项设置
- OCR Provider 启用与默认项设置
- API Key 配置入口
- 本地 HTTP 开关与端口配置
- 系统权限状态查看

### 8.2 V1 不做
- 高级主题配置
- 多工作区配置
- 多配置集切换
- 云同步

## 9. 通用完成定义
某功能在 V1 中“完成”，必须同时满足：
- 有明确触发方式
- 有明确成功结果
- 有明确失败提示
- 有一致的状态流转
- 能写入必要配置
- 能被测试覆盖到主链路

## 10. V1 总体非目标
- 插件系统
- TTS
- 生词本
- 复杂历史管理
- 剪贴板监听
- 跨平台完整适配

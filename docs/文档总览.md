## 2. 文档职责
- [产品规划](./产品规划.md)：定义做什么、不做什么、版本边界和核心场景。
- [技术选型](./技术选型.md)：定义用什么做，以及为什么这样选。
- [需求边界](./需求边界.md)：把 V1 各功能的输入、输出、失败条件和非目标写死。
- [任务状态机](./任务状态机.md)：统一任务生命周期和 UI 状态切换规则。
- [Provider 契约](./Provider契约.md)：统一翻译和 OCR Provider 的接口、字段和错误模型。
- [HTTP API 契约](./HTTP-API契约.md)：统一本地 localhost API 的路由、请求、响应和错误格式。
- [macOS 权限模型](./macOS权限模型.md)：定义辅助功能、截图权限和失败引导。
- [配置模型](./配置模型.md)：定义 Store、Keychain、默认值、迁移和配置边界。
- [错误处理规范](./错误处理规范.md)：定义错误码、展示、日志、重试和中断策略。
- [项目结构约定](./项目结构约定.md)：定义目录结构、依赖方向、命名和测试位置。
- [里程碑与验收](./里程碑与验收.md)：定义阶段拆分、验收标准和非功能指标。
- [翻译 API 清单](./翻译API清单.md)：收敛候选翻译服务官网与定价入口，作为 Provider 对接前的统一索引。

## 3. 当前文档体系关注点
- `macOS` 为正式支持平台，`Windows` 为实验支持平台
- `V1 只保留 4 个高频入口`
- `V1 保留本地 HTTP 调用，但不做插件系统`
- `设置主窗口 + 独立结果工作区` 是当前 UI 结构
- `失败必须显式暴露，不做静默降级`
- `平台复杂度集中到 Swift Helper，业务复杂度集中到 Rust`

## 4. 使用方式
- 做需求取舍时，先看 [产品规划](./产品规划.md) 和 [需求边界](./需求边界.md)。
- 做技术实现时，先看 [技术选型](./技术选型.md)、[Provider 契约](./Provider契约.md) 和 [项目结构约定](./项目结构约定.md)。
- 做翻译服务选型和账号开通时，先看 [翻译 API 清单](./翻译API清单.md)。
- 做 UI 状态和任务编排时，先看 [任务状态机](./任务状态机.md)。
- 做本地调用和脚本集成时，先看 [HTTP API 契约](./HTTP-API契约.md)。
- 做权限、错误、配置时，分别看对应专题文档，不在代码里临时决定。

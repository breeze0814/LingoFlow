# 轻量桌面翻译器里程碑与验收
## 1. 目标
把 V1 拆成可以顺序推进的阶段，避免一开始并行啃所有高风险能力。

## 2. 里程碑
### M0 工程初始化
交付物：
- Tauri 2 + React 19 + TypeScript + Rust 工程可运行
- 基础目录结构落好
- 文档目录完整

验收：
- 本地可以启动桌面应用
- 基础命令可执行

### M1 核心编排与直连翻译
交付物：
- orchestrator 初版
- TranslateProvider 抽象
- `POST /translate`
- 主结果面板基础 UI

验收：
- 直接输入文本可完成翻译
- 成功、失败、取消状态可区分

### M2 输入翻译与设置页
交付物：
- 输入面板
- 快捷键设置
- Provider 设置
- 配置持久化

验收：
- 快捷键可唤起输入翻译
- 配置重启后仍存在

### M3 划词翻译
交付物：
- Swift Helper 可读取前台选中文本
- 辅助功能权限检查
- `GET /selection_translate`

验收：
- 有权限时可完成划词翻译
- 无权限时给出明确引导

### M4 截图 OCR 与截图翻译
交付物：
- 截图链路
- Apple Vision OCR
- `GET /ocr_recognize`
- `GET /ocr_translate`

验收：
- 截图 OCR 成功
- 截图翻译成功
- 取消截图返回 `cancelled`

### M5 稳定性收口
交付物：
- 错误码收敛
- 日志字段收敛
- HTTP API 契约对齐
- 基础测试补齐

验收：
- 主要错误都有稳定错误码
- 单元测试和集成测试覆盖主链路

## 3. 阶段顺序原则
- 先打通直连翻译，再碰平台能力。
- 先打通输入面板，再做选中文本读取。
- 先打通截图 OCR，再做截图翻译。

## 4. V1 总体验收
- 划词翻译可用
- 输入翻译可用
- 截图 OCR 可用
- 截图翻译可用
- 本地 HTTP API 可用
- 设置项可保存
- 权限失败有明确提示

## 5. 非功能指标
- 冷启动目标：`< 2s`
- 已启动状态下输入翻译面板唤起：`< 300ms`
- 直连文本翻译中位耗时：`< 2s`，不含第三方服务严重抖动
- 本地 OCR 中位耗时：`< 1.5s`
- 常驻空闲内存目标：`< 250MB`
- 安装包体积目标：尽量控制在 `100MB` 内

这些指标用于约束“轻量”目标，不是营销指标。

- 简单重试

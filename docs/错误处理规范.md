# 轻量桌面翻译器错误处理规范
## 1. 目标
统一错误码、UI 提示、日志和重试边界，贯彻“失败明确暴露”的原则。

## 2. 错误分类
- `permission_denied`
- `no_selection`
- `empty_input`
- `user_cancelled`
- `provider_not_configured`
- `provider_not_enabled`
- `provider_auth_error`
- `provider_network_error`
- `provider_timeout`
- `provider_rate_limited`
- `provider_invalid_response`
- `ocr_empty_result`
- `http_invalid_request`
- `http_port_in_use`
- `internal_error`

## 3. 统一错误结构
```json
{
  "code": "provider_timeout",
  "message": "Translate request timed out",
  "retryable": true
}
```

## 4. UI 展示规则
- 所有终态失败都必须有明确错误文案。
- 用户取消不显示红色错误态。
- 权限错误必须带功能名和权限名。
- 配置错误必须告诉用户去设置页修正。
- 内部错误允许展示简短文案，同时写详细日志。

## 5. 日志规则
- 每个错误必须带 `task_id`。
- Provider 错误必须带 `provider_id`。
- 权限错误必须带 `permission_type`。
- HTTP 错误必须带路由和状态码。

## 6. 重试规则
- V1 只允许用户显式点击重试。
- V1 不做自动换 Provider。
- V1 不做静默自动重试。
- `retryable=false` 的错误不展示“重试”主按钮。

## 7. 中断规则
以下情况必须直接中断任务：
- 缺少必要权限
- 输入为空
- 没有选中文本
- Provider 未配置
- OCR 返回空内容且任务类型为 `ocr_translate`

## 8. HTTP 映射规则
- 参数错误映射 `400`
- 状态冲突映射 `409`
- 内部异常映射 `500`
- 用户取消保留 `200`，但 `status=cancelled`

## 9. 禁止事项
- 把失败包装成成功空结果
- 自动切换别的 Provider 掩盖失败
- 只写日志不反馈用户
- 使用模糊文案如“未知错误”作为默认提示

# 轻量桌面翻译器 macOS 权限模型
## 1. 目标
明确哪些功能依赖系统权限、缺失权限时如何失败、如何向用户解释。

## 2. V1 相关权限
- `辅助功能权限`：用于读取前台应用选中文本。
- `屏幕录制权限`：用于截图 OCR 和截图翻译。

V1 不额外引入独立 OCR 权限。Apple Vision 在拿到图像后执行识别，不单独申请系统授权。

## 3. 功能与权限映射
- 划词翻译：依赖辅助功能权限。
- 输入翻译：不依赖上述两项权限。
- 截图 OCR：依赖屏幕录制权限。
- 截图翻译：依赖屏幕录制权限。
- 本地 HTTP `selection_translate`：依赖辅助功能权限。
- 本地 HTTP `ocr_*`：依赖屏幕录制权限。

## 4. 权限状态
- `unknown`：尚未检查。
- `granted`：已授权。
- `denied`：明确未授权或被系统限制。

## 5. 启动时策略
- 启动时允许读取权限状态，但不主动弹权限请求。
- 首次真正用到相关能力时再引导授权。
- 不因为缺少某项权限而阻止整个应用启动。

## 6. 功能触发时策略
### 6.1 划词翻译
- 若缺少辅助功能权限，任务直接失败。
- UI 明确提示需要开启辅助功能权限。
- 提供打开系统设置的引导按钮。

### 6.2 截图类功能
- 若缺少屏幕录制权限，任务直接失败。
- UI 明确提示需要开启屏幕录制权限。
- 不尝试用剪贴板或其他方式兜底。

## 7. 失败展示原则
- 提示要具体到权限名称。
- 提示要具体到受影响功能。
- 不能只写“操作失败”。
- 用户主动取消系统授权流程时，不显示假成功。

## 8. 调试与日志
- 每次权限检查都写调试日志。
- 权限失败日志必须包含：
- `permission_type`
- `feature`
- `task_id`

## 9. 系统集成原则
- 平台权限检查和平台调用都放 `Swift Helper`。
- Rust 只接收统一权限状态和错误结构。
- React UI 只接收可展示文案和动作建议。
- 当前实现支持在设置页手动刷新权限状态。

## 10. V1 非目标
- 不做复杂权限诊断工具。
- 不做自动修复权限。
- 不支持绕过系统权限模型。
